# audio prompt 模板

仅 chat 模式 (describe / ask) 用 prompt; asr 模式纯转录不需要。

## 1. 通用概述 (describe 默认)

```
请概述这段音频的核心内容、说话人语气与关键信息。
```

进阶 (结构化):
```
分四段概述: (1) 音频时长与场景 (会议/访谈/录音); (2) 说话人数量与角色; (3) 核心话题与关键论点; (4) 整体氛围或情绪。每段 ≤ 80 字。
```

## 2. 摘要 (会议/访谈)

```
这是一段会议/访谈录音。请输出:
- 主题 (1 句)
- 关键论点 (≤ 5 条 bullet)
- 决议或行动项 (action items, 若有)
- 未决问题 (若有)
```

## 3. 说话人区分

```
按说话人分段输出, 用 [Speaker A] / [Speaker B] 标记。
- 不知道真实姓名时按字母顺序编号
- 每段保留 ≤ 50 字关键点, 不逐字
- 多人讨论穿插, 按时间顺序排
```

## 4. 时间戳标注 (近似)

```
按时间区间标注关键内容:
| 起止时间戳 (近似) | 说话人 | 内容摘要 (≤ 30 字) |

时间无法精确到秒时给 "00:00-01:30" 这种近似区间。
```

## 5. 内容审核类问答 (ask)

```
这段音频中有没有提到以下内容? 一次性给出 yes/no + 引用片段:
- 个人隐私 (手机号/身份证)
- 财务数据
- 商业机密
```

## 6. 反模式

- 不要 "完整逐字转写" 当 describe — 那是 asr 模式的事 (调 transcribe)
- 不堆形容词描述音色 — 模型主观判断不稳
- 不要求"识别音乐流派" — 模型对纯音乐分类弱
- chat 模式上传的音频建议 ≤ 5min, 长音频先切片或用 asr 转录后再喂 LLM 总结
