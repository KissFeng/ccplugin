# prompt 模板库

## 1. 通用描述 (describe 默认)

```
请详细描述这张图片的内容、构图、风格和关键元素。
```

进阶版 (主体 + 风格 + 镜头 + 情绪):
```
分四段描述这张图: (1) 主体与动作; (2) 场景与背景; (3) 视觉风格、色调、构图;
(4) 整体氛围与可能的应用场景。每段 ≤ 80 字。
```

## 2. OCR / 文字提取

```
把图中所有可见文字按原始版式输出, 用 markdown 表达。
- 标题用 # / ##
- 表格用 | 分隔
- 列表用 - 或 1.
- 多语言保持原文, 不翻译
- 模糊或被遮挡部分用 [?] 标记
```

## 3. 视觉问答 (ask 模式)

输入 question 即 prompt 本身, 例如:
- "图里有几个人?"
- "这是什么品牌的车?"
- "图中错误代码是什么? 哪一行?"
- "这张图想表达的核心信息是什么?"

## 4. 表格识别

```
图中是一张表格。请按以下要求输出:
1. 输出标准 markdown 表格
2. 第一行是表头
3. 数字保留原始小数位
4. 空单元格用 -
5. 不输出表格以外的任何说明
```

## 5. 海报 / 卡片字段抽取 (extract 模式辅助 prompt)

schema 文件已驱动 prompt, 通常不需要额外。但对于复杂海报可写补充 prompt
作为 `--prompt` (ask 模式)：

```
这是一张活动海报。抽取以下字段, 输出纯 JSON:
- title (活动标题)
- date (YYYY-MM-DD, 若有时间段取开始日期)
- location (地点)
- speakers (list of string)
- registration_url (报名链接)
- highlights (3 句核心卖点)
找不到的字段用 null。
```

## 6. 代码截图 review

```
图中是代码截图。请:
1. 复刻代码内容 (用对应语言代码块, 保留缩进)
2. 指出明显的 bug 或反模式 (≤ 3 条)
3. 给出修复建议 (diff 风格)
```

## 7. 图表 (chart) 数据反推

```
图中是一张数据图表 (折线 / 柱状 / 饼图)。请:
1. 识别图表类型
2. 提取轴标签与单位
3. 用 markdown 表格列出数据点 (尽量精确, 估算的标注 ~)
4. 一句话总结趋势
```

## 反模式

- 不写 "请尽可能详细" — 给具体字段反而稳
- 不堆形容词 "细致 / 完整 / 全面" — 模型会过度发挥
- 不让模型同时干两件事 (描述 + 评价) — 拆两次 call
- 不让模型给"评分" / "打分" — 主观稳定性差
