# video prompt 模板

## 1. 通用描述 (默认 describe)

```
请详细总结这段视频的关键内容、场景与人物动作。
```

进阶 (按结构):
```
分四段总结: (1) 场景与时长概览; (2) 主要人物/物体出现顺序; (3) 关键动作/事件; (4) 整体主题。每段 ≤ 100 字。
```

## 2. 章节切分

```
观察这段视频, 按内容主题切分章节。输出 markdown 表:
| 章节 | 起止时间戳 (近似) | 主题摘要 (≤ 30 字) |

时间戳估算到秒, 无字幕时按视觉变化推断。
```

## 3. 字幕/口述复刻

```
转写视频中所有可见文字与可推断的口述内容。
- 屏幕文字按出现顺序, 用 markdown 标号
- 口述用引号, 按说话人 (若可区分) 分行
- 不确定的标 [?]
```

## 4. 教程视频结构化提取 (extract schema)

```json
{
  "title": "string",
  "intro_summary": "string",
  "steps": [{"step_no": "integer", "action": "string", "timestamp_seconds": "number"}],
  "tools_mentioned": ["string"],
  "duration_seconds": "number"
}
```

## 5. 监控/事件视频 VQA

```
图里在什么时间发生了什么异常事件? 用以下格式答:
- 时间戳: HH:MM:SS
- 事件: <描述>
- 位置: <画面位置>
```

## 6. 反模式

- 不写 "尽可能详细" — 视频本来信息密度高, 详细容易跑偏
- 不要求模型"理解情感" — 视觉模型对情感推断稳定性差
- 不让模型预测"接下来发生什么" (画面外推不可靠)
- frames 模式时, prompt 提一句"以下是从视频均匀采样的 N 张帧" 帮助模型理解时间维度
