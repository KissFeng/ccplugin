# 编程式调用 + cron

cortex 提供 16 个 bash wrapper (`scripts/cron/*.sh`) 通过 `claude --bare -p` 编程式调用本插件 skill, 实现 daily lint / daily dashboard / daily digest 等周期维护。

## 1. claude CLI 关键 flag

| Flag                                       | 作用                                                                    | cron 必用 |
| ------------------------------------------ | ----------------------------------------------------------------------- | --------- |
| `-p, --print`                              | non-interactive 单次执行                                                | ✓         |
| `--bare`                                   | 关 SessionStart hook + 全局注入 (caveman / cortex 自身), 避免污染       | ✓         |
| `--no-session-persistence`                 | 不写 transcript 到 `~/.claude/projects/`, 防膨胀                        | ✓         |
| `--settings <path>`                        | 显式 settings (推荐 `~/.claude/settings.glm-4.7-flash.json` 廉价 model) | ✓         |
| `--output-format stream-json`              | NDJSON 流, 边跑边过滤                                                   | ✓         |
| `--verbose`                                | 提升事件密度, 失败时回放友好                                            | ✓         |
| `--allowed-tools "Bash Read Glob Grep"`    | 收紧权限, 只读类 cron 用                                                | ✓         |
| `--strict-mcp-config --mcp-config <file>`  | 限定 MCP server (e.g. /dev/null 全清)                                   | 推荐      |
| `--max-budget-usd 0.30`                    | 美元上限, 防 LLM 跑飞                                                   | ✓         |
| `--session-id $(uuidgen)`                  | 显式 session id, 便于追溯                                               | 可选      |
| `--exclude-dynamic-system-prompt-sections` | 移除 cwd/git/env, 提高 prompt cache hit                                 | 推荐      |

完整 flag 表见 `research/01-claude-code-programmatic.md §A`。

## 2. 退出码不可信

`claude` 退出码 `0` ≠ "任务成功"。例:

- `--disallowed-tools Bash` 让 LLM 弃用工具仍 exit 0
- 需必查 `result.is_error` 字段才能确定

cortex 的 `run.sh` 已封装:

```bash
is_error=$(jq -r '.is_error' "$RESULT_FILE")
[[ "$is_error" == "true" ]] && exit 1
```

## 3. 三个标准 cron 任务

### 3.1 daily lint (01:00)

```bash
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/lint.sh \
  --vault /path/to/vault \
  --lang zh-CN
```

只读, 输出 lint summary 到 `~/.cache/cortex/cron/lint-<YYYY-MM-DD>.json`。

### 3.3 daily dashboard (02:30)

```bash
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/dashboard.sh --vault /path/to/vault
```

刷新既有 dashboard 页 (Bases 或 Dataview 自动选择)。

## 4. 通用 wrapper run.sh

```bash
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/run.sh <job-name> -- <prompt> [extra claude flags]
```

提供:

- 锁文件 (`/tmp/cortex-<job>.lock`, flock 或 PID lockfile)
- 超时 (`CORTEX_TIMEOUT=300` 默认)
- 日志 (`~/.cache/cortex/cron/<job>-<YYYY-MM-DD>.log` 与 `.err` 与 `.json`), 7 天轮转
- 退出码: 0 成功 / 1 一般失败 / 2 锁忙 / 3 超时 / 4 配置错

环境变量:

- `CORTEX_VAULT` (必需)
- `CORTEX_LANG` (可选)
- `CORTEX_SETTINGS` (默认 `~/.claude/settings.glm-4.7-flash.json`)
- `CORTEX_DRY_RUN=1` 仅打印命令不执行

## 4.1 共享配置 `~/.cortex/config.json`

cortex 把跨入口的运行参数集中在用户级 JSON, 供 cron / hooks / lint / 一键安装共享同一份事实源。

路径: `~/.cortex/config.json`

Schema (扁平, 全字段可选):

```json
{
	"vault": "/abs/path/to/vault",
	"lang": "zh-CN",
	"settings": "/abs/path/to/claude-settings.json"
}
```

plugin 源码路径硬编码字面量 `~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex` (不存 config, 不可覆盖)。

字段查找优先级 (高到低, 任一命中即停):

| 字段           | 顺序                                                                                |
| -------------- | ----------------------------------------------------------------------------------- |
| `vault`        | `CORTEX_VAULT` env > `config.vault` > `OBSIDIAN_VAULT` env > 报错退出               |
| `lang`         | `CORTEX_LANG` env > vault `_meta/version.json` > `config.lang` > `zh-CN`            |
| `settings`     | `CORTEX_SETTINGS` env > `config.settings` > `~/.claude/settings.glm-4.7-flash.json` |

错误处理: JSON 语法损坏 → 所有消费点 **fail-fast** (stderr 报 "config syntax error at ~/.cortex/config.json: <detail>"); 字段缺失 → 视为未配置, 走后续 fallback。

CLI 管理 (经 wrapper 或直接调脚本):

```bash
~/.cortex/scripts/config.sh get vault                # 读单字段
~/.cortex/scripts/config.sh set lang en-US           # 写单字段 (含路径存在校验)
~/.cortex/scripts/config.sh init --non-interactive \  # 批量写
    --vault /path --lang zh-CN --settings /path/settings.json
~/.cortex/scripts/config.sh validate                 # 校验 JSON + 字段语义
```

或直接 `python3 <plugin>/scripts/cortex_config.py {get|set|init|validate} ...`。

## 4.5 cron 路径

cron daemon 与登录 shell 环境隔离, **不会**继承 `$CLAUDE_PLUGIN_ROOT`。snippet 中 `PLUGIN_ROOT` 硬编码字面量:

```
~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex
```

跨平台一致 (macOS / Linux / Windows Git-Bash 下 `~` 均展开为 `$HOME`)。不接受 `--plugin-root` flag / env 覆盖。

调用:

```bash
bash scripts/install_cron.sh cron     # crontab
bash scripts/install_cron.sh launchd  # macOS launchd plist
bash scripts/install_cron.sh gha      # GitHub Actions workflow
```

GHA snippet 例外: CI 环境用 `${GITHUB_WORKSPACE}/plugins/tools/cortex/...` 占位, 由 Actions runner 注入。

## 5. cron 注册 (cortex-install §7 内联)

cortex-install 末尾询问注册; 也可后续主动跑 `/cortex:install_cron`。

平台映射:

- macOS → `~/Library/LaunchAgents/dev.lazygophers.cortex.<job>.plist`
- Linux → `crontab -l` 追加行 (备份原 crontab) 或 systemd user timer
- CI (`$CI=true`) → 输出 `.github/workflows/cortex-<job>.yml` 让用户复制, 不真写

卸载: 见 cortex-install SKILL.md §卸载提示。

## 6. 常见故障

| 症状                   | 原因                                        | 处理                                                      |
| ---------------------- | ------------------------------------------- | --------------------------------------------------------- |
| exit 0 但 vault 没变化 | LLM 跑了但工具被拒                          | 查 `~/.cache/cortex/cron/<job>.err`; 调 `--allowed-tools` |
| exit 2 (锁忙)          | 上次 cron 没退出                            | `rm /tmp/cortex-<job>.lock` 后排查                        |
| exit 3 (超时)          | LLM thinking 长                             | 增大 `CORTEX_TIMEOUT` 或缩小任务                          |
| exit 1 + is_error=true | API 错 / max-budget 超                      | 看 `.json` 里的 `result` 字段                             |
| 没有 `.json` 输出      | jq 解析失败                                 | 看 `.err` 里的 stderr                                     |
| keychain 弹窗 (macOS)  | 没用 `--bare` 或 settings 缺 `apiKeyHelper` | 强制 `--bare` + settings 配 `ANTHROPIC_API_KEY`           |

## 7. 安全

- cron 任务**永不**用 `--dangerously-skip-permissions`
- 写型任务白名单严格 (`Read Write Edit Glob`, 不含 `Bash` 任意命令)
- 不动 user `~/.claude/settings.json`; 仅写 LaunchAgents/crontab/`.github/workflows`
- 卸载脚本可一键回退
