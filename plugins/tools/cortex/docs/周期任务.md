# 周期任务

本文回答：cortex 推荐的 3 个周期任务是什么、`install_cron.sh` 怎么用、在 macOS launchd / Linux cron / GitHub Actions 上分别怎么写。
适用读者：想让 cortex 在后台自动维护 vault 的运维。

## 推荐 4 个任务

| 任务            | 频率           | 命令                                                                                                  | 说明                                            |
| --------------- | -------------- | ----------------------------------------------------------------------------------------------------- | ----------------------------------------------- |
| lint            | daily 01:00    | `bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/lint.sh`        | autofix 循环修复 17 类规则                      |
| dashboard       | daily 02:30    | `bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/dashboard.sh`   | 重渲 index.md / hot.md / canvas                 |
| consolidate     | daily 03:00    | `bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/cron/digest.sh` | log/session 五阶段处理 (读+析+处+更新+清理)     |

四个任务都通过 wrapper bash 调 `claude -p "/cortex:<name>"` 触发, 需先装 Claude Code CLI (官方安装渠道)。

## install_cron.sh 用法

cortex 不自动写用户 crontab / launchd, **仅打印 snippet 供用户复制**。

```bash
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/install_cron.sh cron      # 默认: Linux/macOS cron
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/install_cron.sh launchd   # macOS launchd plist
bash ~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/install_cron.sh gha       # GitHub Actions workflow
```

输出直接复制到对应位置即可。

## Linux / macOS cron

`bash install_cron.sh cron` 输出：

```bash
# ===== cortex cron snippet (Linux/macOS cron) =====
# 复制以下行到 'crontab -e':

# daily lint with autofix at 01:00
0 1 * * * /usr/bin/env python3 "~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/lint/run.py" --vault "${VAULT}" --fix > "$HOME/.cache/cortex/lint-$(date +%Y%m%d).json" 2>&1

# daily dashboard refresh at 02:30 (claude CLI required)
30 2 * * 0 claude -p "/cortex:dashboard refresh all" >> "$HOME/.cache/cortex/dashboard.log" 2>&1
```

注意：

- `~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex` / `${VAULT}` 在脚本生成时已被替换成实际路径。
- 用 `crontab -e` 粘贴, `crontab -l` 验证。
- macOS 默认 cron 受 SIP 限制, 建议用 launchd。

## macOS launchd

`bash install_cron.sh launchd` 输出 plist：

```xml
<?xml version="1.0" encoding="UTF-8"?>
<plist version="1.0">
<dict>
  <key>Label</key>
  <string>dev.KissFeng.cortex.daily-lint</string>
  <key>ProgramArguments</key>
  <array>
    <string>/usr/bin/env</string>
    <string>python3</string>
    <string>~/.claude/plugins/marketplaces/ccplugin-market/plugins/tools/cortex/scripts/lint/run.py</string>
    <string>--vault</string>
    <string>${VAULT}</string>
    <string>--fix</string>
  </array>
  <key>StartCalendarInterval</key>
  <dict>
    <key>Hour</key><integer>1</integer>
    <key>Minute</key><integer>0</integer>
  </dict>
  <key>StandardOutPath</key>
  <string>${HOME}/.cache/cortex/lint.log</string>
  <key>StandardErrorPath</key>
  <string>${HOME}/.cache/cortex/lint.err</string>
</dict>
</plist>
```

操作：

```bash
bash install_cron.sh launchd > ~/Library/LaunchAgents/dev.KissFeng.cortex.daily-lint.plist
launchctl load ~/Library/LaunchAgents/dev.KissFeng.cortex.daily-lint.plist
launchctl list | grep cortex   # 验证
```

派生 dashboard plist：

- `StartCalendarInterval` 加 `Weekday=0` (周日), 改 `Hour` / `Minute`

## GitHub Actions

`bash install_cron.sh gha` 输出 workflow：

```yaml
name: cortex-cron

on:
  schedule:
    - cron: "0 1 * * *" # daily lint
  workflow_dispatch:

jobs:
  lint:
    if: github.event.schedule == '0 1 * * *'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.11" }
      - name: cortex lint
        run: python3 lint/run.py --vault . --fix > lint-report.json
      - uses: actions/upload-artifact@v4
        with: { name: lint-report, path: lint-report.json }
